HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta element defines metadata like the character set, description, keywords, author, and viewport
  • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • inline CSS — for styling an element directly in HTML; useful for quick and permanent changes; less flexibility; for individual situations
  • internal CSS style sheet — embed an entire stylesheet directly in the HTML file using a style element in the head element
  • external CSS style sheet — kept in a separate file, which gets link in the HTML’s head

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git commit: finalizes code and merges with the main branch, be sure to add message to it. Ex. git commit -m "add starter code"
  • pull: receive branch’s modifications into the local environment
